The specs

  • Take a list of hex colours, with or without descriptive names, and sort them by hue
  • Take a default selected colour which will be displayed
  • When the default colour is hovered over, a palette of swatches sorted by hue will fade in allowing the user to hover over each swatch in order to change the default selected colour
  • The palette of swatches will be paginated to avoid showing too many swatches on screen at one time. The user will be able to click left or right to move through the entire palette of hues
  • Emit events. It will emit two events: one when a colour is hovered over, another when a colour is clicked on. Both events will pass the hexcode of the colour to any listeners.

Extending the jQuery object

How many times have you used the dollar sign $ in jQuery but never stopped to think about what it actually is? First of all, it’s shorthand for the jQuery object. Therefore calling $() and jQuery() is exactly the same. Secondly, what are you doing when you’re calling $()? Well, you’re calling the constructor function of the jQuery object. This constructor function returns an HTML element that is extended with all the magic and sparkle that jQuery’s methods provide.

The jQuery object exposes an object accessible via $.fn or jQuery.fn, where all its methods, such as .html() and .append() live. This is jQuery's equivalent of the prototype object, and it‘s the place where we‘ put our new plugin function. So we need to start by extending the $.fn object.
